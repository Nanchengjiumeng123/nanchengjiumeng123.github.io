版本以及下载地址

版本选择

5.5.3

因为spring boot选择了2.0.6.RELEASE,spring boot2.X版本对应es中的5.5.X版本

下载地址

https://www.elastic.co/cn/downloads/past-releases/elasticsearch-5-5-3

部署

文件路径创建

1.在指定位置解压es压缩包

tar -zxvf *
创建数据以及日志位置

数据存储位置

mkdir /data/es/data

日志存储位置

mkdir /data/es/logs
修改配置

修改jvm.options

配置jvm占用内存大小,默认是2g,测试机器是4g的内存,但是同时其他进程需要占用,所以就只分配1g

-Xms1g
-Xmx1g
2.修改elasticsearch.yml

集群名称:

整个集群中所有节点的集群名是相同的

cluster.name: elasticsearch_statistics_dev
节点名称

只要保证集群中每个几点不同即可(最好还能知道是哪个服务器)

node.name: node22
数据存储

path.data: /data/es/data
日志存储

path.logs: /data/es/logs
ip地址

network.host: 192.168.0.22
端口号

http.port: 9200
集群配置

集群中服务器ip地址,可以写多个,有节点启动时会向里面的ip发送消息,获取集群其他节点的信息

discovery.zen.ping.unicast.hosts: [“192.168.0.22”, “192.168.0.23”,”192.168.0.24”]
master重新选举,避免脑裂(这个也是可以通过es的api来修改)

一般就是n/2 + 1 n是节点数

discovery.zen.minimum_master_nodes: 2
避免频繁的shard中数据重新划分

#在启动多少个节点之后才会考虑shard中数据重新划分
gateway.recover_after_nodes: 2
#集群中总节点数
gateway.expected_nodes: 3
#等待超时时间,如果超过时间才会重新划分shard,前提是启动的节点数超过上面的设置
gateway.recover_after_time: 5m
启动es之前需要的操作

创建一个普通用户,root用户不允许es
adduser esuser
passwd esuser
赋予esuser用户es文件夹以及数据存储文件夹权限
chown -R esuser /usr/local/elasticsearch-5.5.3/
chown -R esuser /data/es/
3.修改系统文件

vi /etc/security/limits.conf

在文件最后，增加如下配置：

esuser hard nofile 65536
esuser soft nofile 65536

  •     soft     nproc      4096

  •     hard     nproc      4096

4.切换到esuser用户

su esuser

启动es

bin/elasticsearch
5.开启9200端口号访问

查看端口是否开放

netstat -nltp | grep 9200
/sbin/iptables -I INPUT -p tcp –dport 9200 -j ACCEPT
在浏览器可以访问,访问到即成功

可能会出现的问题

max virtual memory areas vm.max_map_count [65530] is too low,

解决:

切换到root用户，在/etc/sysctl.conf文件最后添加一行
vm.max_map_count=655360

添加完毕之后，执行命令：
    sysctl -p
